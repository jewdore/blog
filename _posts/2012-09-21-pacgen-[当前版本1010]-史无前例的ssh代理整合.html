在进入正文之前, 先给大家科普一下, 什么是PAC(Pacgen这个工具名展开来便是PACfileGenerator, 当然, 自从它集成代理工具以来, 它就不止是generator了).

PAC全称Proxy Automatic Configuration, 代理自动配置, PAC文件实际上是一个含有某些特定名称函数的JS脚本, 但对于多数人来说, 这些不重要, 重要的是PAC能做什么!

如果大家使用过代理爬墙, 可能多数人的情况是, 开始爬墙, 则浏览国内网站会很慢, 于是在墙外干完该干的事, 又急急忙忙关掉代理. 但在PAC的帮助下, 可以让你在浏览被墙网站时, 使用代理, 而其他网站则直接连接. 于是在同一个浏览器(推荐Chrome), 同一个时间, 你就可以同时浏览墙内墙外的网站了.

正文开始.

前段时间用HTA (HTML Application)做了个, 但是在电脑上兼容性不够好, 而且功能也很有限, 于是用C#重写了个. 分析了注册表中的代理设置, 因为是binary数据, 所以有点麻烦, 然后直接修改, 所以现在使用Pacgen相当方便, 除了安装, 只需要输入SSH服务器, 账号, 密码, 再点击Start即可. 不用其他设置即可在Windows下的IE, Chrome, Safari(Safari未实测, 理论上应该可以)使用, FF只需稍微修改设置即可.

下面是Pacgen的界面.

所以教程什么的, 就真的很难写了, 太傻瓜了. 不过顺带说下. 因为Pacgen是基于 .NET Framework 3.5 开发的, 所以需要安装这个框架, Windows 7已经自带了此框架, 所以不用安装了, 部分XP用户或者Vista用户, 如果没有安装的话, 则会提示安装.

图标还好看吧, 简约风, 安装包https://raw.github.com/vilic/pacgen/master/setup.exe如果之前有安装Pacgen (非HTA版本, 就是长得跟现在这个很像的), 最好先到控制面板中卸载掉.

安装完成后, 在Server一栏填写SSH的服务器地址, 通常会是一个域名或者IP, 比如aaa.bbb.com, 请在SSH账号提供商哪里找到相关信息. 然后在Username和Password处分别填上SSH服务器的用户名和密码, 确认无误后点击Start. 一般来说, 设置这三个内容就已经足够使用Pacgen了, 但后面还有一些相对高级的选项, 如果你不是很懂, 最好是不要动. 但我还是稍作解释. 不过有些SSH服务器提供的端口可能不是22, 请参见下面的2.

/* 可忽略内容开始 */

1. Start Pacgen proxy with Windows, 开机启动Pacgen并连接代理, 默认勾选.2. Server (服务器)后面的数字, 默认是22, 可以在SSH账号提供商处找到, 一般都是22.3. Proxy port (代理端口), 有两个数字, 分别是SOCKS代理端口和HTTP代理端口, 默认是1107和9257, 这里取1025~65534之间的任意数字, 但是不能被其他程序占用, 如果Pacgen可以正常使用则无需更改.4. Use online rules generated by Autoproxy, 使用储存在网上的Autoproxy规则, 这些规则里包含了大多数被墙的网站, 默认勾选, 有这些规则基本就能正常爬墙了.5. Use custom rules, 使用自定义规则, 有时候也会有被墙的网站没有被收录到Autoproxy的规则中, 这个时候可以自己添加规则来对相应网站爬墙, 当然, 有时候也可以将一些容易被 "重置", "中断" 的网站加上去, 或者是想通过代理提高访问速度的网站, 总之是你想使用代理的网站.6. Automatic PAC proxy configuration, 自动PAC代理配置, 这个其实有点含糊, 在PAC文件中, 有一个格式如 "METHOD PROXYHOST:PORT" 的字符串, 用来告诉系统应该如何访问对应的站点, 比如 "PROXY 127.0.0.1:9257", 则是告诉系统, 使用HTTP代理, 代理地址是127.0.0.1, 端口是9257. 或者也可能是 "DIRECT", 标示不使用代理, 直接访问. 如果勾选这个选项, Pacgen会自动生成对应的字符串, 并通过其他手段稍加优化, 让IE使用HTTP代理的同时(IE不支持SOCKS5代理), Chrome能使用SOCKS5代理. 如果去掉勾选, 则不再区分, 统一使用下面文本框中设置的值.

/* 可忽略内容结束 */

片刻之后(这段时间Pacgen内置的Plink会连接SSH服务器并转发SOCKS端口), Pacgen提供的代理应该就能够使用了, 经测试, Google Chrome下, Pacgen的改动是实时生效的, 而系统自带的Internet Explorer则需要重新启动. 在Firefox下, 需要把代理设置设置为使用系统设置.

在安装过程中, 如果防火墙有提示, 请一律允许, 并记住操作. Pacgen默认会随系统启动, 这样以后打开电脑, 便自动爬墙了, 方便吧~

最后再说两个问题, 一个是SSH账号, 如果你不嫌麻烦的话, 也可以找一些免费的SSH账号, 但其实网上有很多SSH账号提供商的收费账号也不是很贵, 我现在用的这个是5元每月, 45元每年, 另外还可以按周/季付款.

注册地址http://www.sshcenter.info/

另外还有http://blog.onlybird.com/(欢迎补充及推荐)

另一个是自定义规则(在Rules标签下), 一般一个网站上的内容会不一定只来自一个域名, 所以为了保证其可用性, 添加规则的时候, 比如www.xxx.com, bbs.xxx.com, 我们能看出这个网站的 "顶级域名" 是xxx.com, 于是在填写规则的时候, 就应该写上xxx.com, 而不是www.xxx.com或者bbs.xxx.com. 当然, 如果你知道这些意味着什么, 就不必管我说的了.
